9 injection patterns catalogued

Every non-trivial system reaches a moment where its designers must answer a question: how do users inject their own rules?

This is the engineering problem that separates a script from a platform. A script does one thing. A platform provides seams — deliberate openings in the architecture where custom logic can be inserted without modifying the core. These seams have names: hooks, middleware, plugins, callbacks, decorators, event buses, strategy patterns, pipeline steps.

The Taxonomy of Extensibility

Across Ted's 33 projects, every extensibility pattern falls into one of three families. Interception patterns (hooks, middleware, interceptors) let custom logic wrap or modify behavior at predefined points. Composition patterns (plugins, pipelines, strategy) let users assemble behavior from swappable components. Reaction patterns (callbacks, event buses, observers) let custom logic respond to events without the core knowing what will respond.

The choice between these families is never purely technical. It encodes assumptions about who the user is and how much they should be trusted. A hook says: "I'll call you at this exact moment." A plugin says: "Register yourself and I'll discover you." An event bus says: "I don't even know you exist, but you can listen."

From Games to Knowledge Systems

The MTG Draft Overlay needed card evaluation logic that changed with every set release — a classic strategy pattern. The Alchemy Board Game needed transmutation rules that players could discover and chain — a pipeline. The Tarot Engine needed spread layouts that occultists could customize — a configuration-driven template system. Dreambase itself is built on Flask routes, Jinja2 templates, and sidecar JSON files — three layers of injection.

The Meta-Pattern

The deepest lesson from cataloguing these patterns: extensibility is a form of humility. Every hook, every plugin API, every event bus is an admission that the designer doesn't know everything. The best systems are the ones that make it easiest to inject logic their creators never imagined.

9 Injection Patterns Across 33 Projects

Interception

Hooks

Fundamental

Named points in execution where custom functions are called. The simplest injection pattern: "I'll call your function here."

Flask's before_request and after_request. Git hooks. Claude Code's pre/post-tool hooks. WordPress's entire ecosystem.

Used in: Dreambase, Claude Code skills, Tarot Engine
Interception

Middleware

Intermediate

A chain of functions that each transform a request/response, passing it to the next. The Russian doll of web engineering.

Express.js middleware stacks. Flask's WSGI middleware. The MarxistTradition site's auth + logging + CORS chain.

Used in: MarxistTradition, MarxistPortal, MTGDraftOverlay
Interception

Decorators

Intermediate

Functions that wrap other functions, adding behavior without modifying the original. Python's @route is the most famous example.

Every Flask route in Dreambase is a decorated function. The SMS analysis pipeline uses decorators for caching and retries.

Used in: Dreambase (app.py), SMS Archaeology, Claude Data Tools
Composition

Plugins

Advanced

Self-registering modules that extend a host application. The host discovers plugins at runtime — maximum decoupling.

The Chiptune Synth's waveform plugins. MTG Draft Overlay's card evaluator plugins (one per set). VS Code extensions as a design model.

Used in: Chiptune Synth, MTGDraftOverlay, Draft Academy
Composition

Pipelines

Fundamental

Sequential processing steps where each step transforms data and passes it forward. Unix pipes made architectural.

Megabase's ingestion pipeline: parse → normalize → insert → index → summarize. Each step is a standalone script.

Used in: Megabase (7 ingestors), SMS Archaeology, Twitter Archive, Capital Interpreter
Composition

Strategy Pattern

Intermediate

Interchangeable algorithms behind a common interface. The caller picks which strategy to use at runtime.

MTG Draft Overlay's card rating algorithms (one per format). The Alchemy Board Game's scoring functions (points vs. transmutation chains).

Used in: MTGDraftOverlay, Alchemy Board Game, Autobattler Proto
Composition

Configuration / Templates

Fundamental

Behavior defined in data files (JSON, YAML, Jinja2) rather than code. The least-code injection pattern.

Dreambase's sidecar JSON files. The Tarot Engine's spread definitions. The Golden Dawn Curriculum's grade configurations.

Used in: Dreambase (sidecars), Tarot Engine, Golden Dawn Curriculum, Hermetic Reading List
Reaction

Callbacks

Fundamental

Functions passed as arguments, called when an event occurs. The building block of async programming.

JavaScript event handlers in every Dreambase template. The Interactive Platonic Solid Viewer's rotation callbacks. Pygame event loops.

Used in: All web templates, Kabbalah Tree Viz, Bubble Prototype, Mancala
Reaction

Event Bus / Observer

Advanced

A central dispatcher that routes events to registered listeners. The core decouples completely from the extensions.

The Electron IPC bus in MTG Draft Overlay. The Memory Palace's Three.js scene event system. DOM CustomEvents in Dreambase tabs.

Used in: MTGDraftOverlay (Electron IPC), Memory Palace, Overlay framework

How the approach to extensibility evolved

PHASE 1: SCRIPTS
Everything in one file
Early projects (build_chats.py, game_ideas.py) were monolithic scripts. No extensibility needed because no one else would use them. The "injection point" was editing the source code.
PHASE 2: CONFIGURATION
Behavior in JSON, not code
The Tarot Engine and Golden Dawn Curriculum moved definitions to JSON files. Users could add spreads or grades without touching Python. First real separation of logic from content.
PHASE 3: PIPELINES
Composable processing steps
Megabase's ingestion architecture: 7 independent scripts that each parse a source and feed a shared database. Each ingestor is a pipeline step. Adding a new source means writing one new script — zero changes to existing code.
PHASE 4: SIDECAR PATTERN
LLM-generated content alongside code
Dreambase's sidecar JSON files let ChatGPT-generated narratives, timelines, and quotes live next to templates without being embedded in them. The code defines the structure; the sidecar provides the content.
PHASE 5: SKILL SYSTEM
Self-describing, invocable units
The PKD skill system (33 slash commands) is the most sophisticated injection pattern: each skill is a self-contained markdown file with a trigger condition, instructions, and arguments. Claude Code discovers and executes them at runtime. The human registers intent; the system finds the matching logic.
PHASE 6: AGENTIC SWARMS
Agents as injectable workers
The Eldritch Swarm architecture: define agent roles, their inputs/outputs, and let them run in parallel. Each agent is injected into a workflow slot. The orchestrator doesn't know what the agents will do — only what they'll produce.
